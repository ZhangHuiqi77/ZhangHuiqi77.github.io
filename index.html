But two awardees for the Nobel Prize in Chemistry indicated a new paradigm of scientific findings. Demis Hassabis and John M. Jumper are from Google DeepMind. Their work is not a small belief. In contrast, it is a goal that many researchers are dreaming. They are not alone on their way. They attacted a huge amount of resources, money, humans to create AlphaFold, the wonderful protein struction prediction tool. Also, it only took them 4 years from publishing their work to getting the Nobel Prize. These means a lot. I want to say is that the era is changing. Now, the paradigm of scientific finding is more open, bigger and maybe without a hypothesis.

But if I focus on the common features of these awardees. They all have a big dream and a specific target. Then what’s next? If I have no technical limitation, what do I want to do? I think the most cool thing is … Imagine that by dialing, people entity can transit in any time to any space. Forget about the dimension of time first, if we want to transit people in any space fast, there are these steps as I image:

  1. the human body proteins info need to be decode and encode to a new coding system.

  2. the new code can be transfer in wireless and fast.

  3. the new code need to be translated back to biology language and used to print out or synthesis protein in seconds.

Decoding of human’s biological info is a huge work. This includes the elements (cells, neurons, chemical stuff) and all connections. Even only sequencing a small amount of cells may need few days to finish for the most advanced sequencing machine and it follows a complicated procedure. Additionally, some protein interaction and the structure is far from described. – No wonder why the multiple omics sequencing is so popular these years. Even though we can only ‘send out’ a C elegans, it is a breakthrough.

If seems only the second step is doable, by internet or radio waves. Biology code can be transfer to silicon code already. Maybe the transfering system can get optimized.

Fast synthesis of protein and other biological elements, or 3d printing an entity is another big challenge.

python environment - conda ipython and spyder

Posted on 2024-01-18 In Technology

Basically, conda is a tool to manage virtual environments, which can have different versions of python or other packages. ‘Virtual’ is not really virtual. It actually installed the needed packages in different path. ipython is an interactive python terminal, which is based on python interpreter (can be understanded as python.exe in base or one vertual environment). spyder is a platform to write script, run code in console, check variables in the workspace, etc.. It also runs based on a python interpreter and specific environment. Only need to install spyder in the base, then install spyder kernels for the environment and edit the python interpreter and start new kernel in the launched spyder.

So the basic steps of setting a new environment is to 1. create env, 2. install packages and ipython. 3. set the python interpreter and console in spyder. Also, after creating environment, need to add the path to the environment variables in the windows “advanced system setting” (C:\Users\zhang\anaconda3\envs\tf\Lib\site-packages, C:\Users\zhang\anaconda3\envs\tf\Lib, C:\Users\zhang\anaconda3\envs\tf\Scripts, C:\Users\zhang\anaconda3\envs\tf\Library, C:\Users\zhang\anaconda3\envs\tf\Library\bin).

These days, I am trying to analyze the behavioral structure based on the DLC tracking data. Some machine learning methods need to be used. Here, just wrote down some experience of packages install. On the one hand, I am using VAME, which do RNN and autoencoder under the pytorch structure and some visualization tool, like umap. On the other hand, I will use tensorflow LSTM. To manage packages, I create Anaconda environments on Windows 10 for both. Actually I did on two PCs, VAME on the GPU and tensorflow on my laptop (only CPU).

# if you tried the same env with similar name before but it does not work, can clear it in conda prompt before try new one.
conda remove --name tf-cpu --all

After create the environment, activate it and install packages. For VAME install, follow the instruction on their github, then edit the error according to error info. Usually some text error, like in the yaml config file, “sklearn” need to change to “scikit-learn”. The same problem also need to be editted in the installation file, here it is ‘setup.py’. Then when it installed umap, the eroor is sth like “importlib.metadata can not be found” and it is point to the umap init file. I checked the importlib.metadata, and it is really not easy to found by eye that for python versions earlier than 3.8, this package is importlib_metadata ( _ there but not .). I did install importlib_metadata and importlib-metadata, so I change the . to _ then it work. I would see some tiny difference of text, because they update new versions and have new dependencies quickly.

For tensorflow installation:

# In conda prompt, create a new environment for tf. 
conda create --name tf
activate tf
conda search tensorflow # search for the available version of tf in anaconda, then choose the version you like
conda install tensorflow=2.10.0
conda install ipython # install ipython for this environment, so I can enter ipython not enter the base env ipython
pip install spyder-kernels==2.2.* # install kernels for using this virtual env in spyder

# after install, can test in the ipython and spyder

print(tf.__version__)

To work on the tf env in spyder, activate the env in conda, then launch spyder by click, and in spyder, go to tools -> preference -> Python interpreter -> change it to the path of python.exe in the virtual env (C:/Users/zhang/anaconda3/envs/tf/python.exe). Then start a new console under this interpreter.

interesting math - proof by contradiction

Posted on 2024-01-14 In Science

proof by contradiction (反证法)
The statement is: There are infinite prime numbers(质数有无限多个)

Assume that there are limited prime numbers，p1, p2, …, pr in order
Then there is a number p = p1p2… * pr + 1,
As pr is the biggest prime number, so p is not prime number. Then p should be rewriten as the multiplication of the existed prime numbers. But the given expression of p show it cannot. Because it is with a +1.
the assumption is wrong.
Proved! there are infinite prime numbers.

There are also an interesting story in this chapter. Holmes and Watson go camping and slept in a tent. At midnight, Holmes wake up and asked Watson to tell him what did her see. Watson looked at the beautiful sky and answered very detailed inference according to her rich knowledge about stars, constellation, astronomy, meteorology and time. She was very proud of herself and asked back what did Holmes see. Holmes wry smiled and said, Waston you idiot, it means our camp was stolen.

I leared that to proof a statement, can create a variable based on the constrain and well-known truth, then try to overthrow the opposite statement. Creat the variable is the important step in the whole process! Can try in other questions.

I am always thinking, mathematic statement are based on the answer is just yes or no. But like in neuroscience, the questions are complex, “high dimensional”. For example, does APC helpful in treating epilepsy? This question is still too broad. To narrow down, we can ask: does APC helpful in rescueing recurrent seizures in the KA model? Then, to find the answer, we need to manipulate the APC, which means there are detailed choices, electrical stimulation, chemogenetics, optogenetics, pharmalogical approaches. We can then narrow down the hypothesis or statement to “inhibiting APC by chemogenetically activating PV interneurons is helpful in rescueing recurrent seizures in the KA model.” Then the animal model can be built and the manipulation can be executed. Then it need a reliable way to assess the recurrent seizures. So we record and analyze the EEG. The results show that seizures get less, which is contradict to the contradictory statement “inhibiting APC by chemogenetically activating PV interneurons is NOT helpful in rescueing recurrent seizures in the KA model.” So we proved our initial statement.
It seems just play with the words of direct proof and proof by contradiction. we don’t even need to create new variables. If we think of it in the another way, we exciting APC and record the seizures and found that exciting APC do make seizure worse. We still cannot say inhibiting APC can improve seizure. It seems narrow down to specific question and direct proof is dominant.

scientist story - Eric Betzig

Posted on 2023-08-18 In Science

This article is from Nature - light people, an interview of Prof. Eric Betzig. He tells us his “true life”, not just pass through. Nobel award, five children, science and technology, father’s company.

The interview is quite interesting.
https://www.nature.com/articles/s41377-023-01205-3#:~:text=Eric%20Betzig%2C%20who%20shared%20the,Bell%20Labs%20before%20turning%2035.

The points hit me the most:

  1. cross-disciplines. But I find that the breakthroughs in science happen when people interact at the interface between very different disciplines that don’t necessarily have a clear connection. To this day, anything that I’ve succeeded within sciences is by getting out of my comfort zone and learning more about some field that has nothing to do with optics, and how optics can be applied to it.

  2. flexibly minded. You have to fill your mind with ideas and thoughts, and creatively think about how different things can be combined. I think that’s why I enjoyed working at Bell Labs so much, because it was a place where cell biologists, neurobiologists, computer scientists, material scientists all worked under the same roof and it was amazing what breakthroughs were made just having people talking to each other. So I want to warn against siloization in scientific research. Don’t just have people studying the same discipline in one institution, or don’t just have people chasing preset goals. Encourage outside thinkers. It’s good to have a goal, but you never know what’s gonna help you get to that goal. And sometimes goals may not have been intelligently chosen, and it can put blinders on you. You have to be flexibly minded in science.

  3. question and different disciplines. I believe to make a good scientific investigator, you need to be open-minded, and have the willingness and ability to talk to many other people of different disciplines, which is the secret to creativity and innovativeness. Knowing the problem is often half of the solution, so you should always have in your mind a set of important and impactful problems that you don’t know how to answer. Then, by putting yourself in a position where you are exposed to different people from different disciplines, you’ll find that creativity will come naturally because if you have the problem in your head, your subconscious starts to think about what you can do to solve this problem.

  4. work hard and success. To succeed, you just have to work really hard, make sure your brain is completely immersed in the problem or a number of problems, and then luck will come. But luck does not determine whether you’ll be successful. Hard work determines whether you’re going to be successful or not. Luck determines what you will be successful at.

  5. exponential curve of intelligence. It’s (the “It” is hard working) the only correlate I know to success. It has nothing to do with intelligence or where you’re working. It’s nonlinear, because the more you know, the more efficient you get in finding out even more. So it goes in pretty much an exponential curve, not a linear curve, with output.

  6. perseverance and quit. Perseverance doesn’t mean staying in the same job forever, nor does it mean focusing on the same problem, because sometimes problems have dead ends. Perseverance means focusing on the science or what motivates you in your career. I will agree that the hardest thing for me has been to know when to quit for exactly that reason because you’re taught: persevere, persevere, persevere. So it was a tough decision to quit Bell Labs and an even tougher decision to leave my dad’s company. At Bell Labs at least I knew that I had taken that near field technique as far as it could go. I knew that I wasn’t going to be able to break the laws of physics. So it wasn’t as difficult a decision to say, “Forget perseverance, I’m quitting there.” Because I felt like I had done everything I could do.

  7. risk takers and truly alive. When I ended my Nobel talk, I dedicated it to all the people who took a risk and failed, because though you don’t hear about them, they are more important because there’re more of them than people who took a risk and succeeded. You can go a whole career having a bunch of good half-baked ideas and never find the key. But I still encourage my students to take risks. When you take a risk, particularly a risk for your career, it usually brings out your best self. When you take a risk and initially fail, you’ll work your absolute hardest to try to succeed and that’s when you find your limits. It’s being alive, about being truly alive as opposed to kind of just passing through life.

  8. luck and scientific question. I felt I was very lucky to go to graduate school in 1982 and pick microscopy as a field. That was a lucky decision because it was right at the time that personal computers were available to automate microscopes, when the CCD chip was just becoming available, when fluorescence was starting to take off, and when lasers were cheap enough that they could go in individual labs. So being at the start of your career in 1980 and choosing microscopy is like being a physicist in 1920 and deciding to study quantum mechanics. That’s the way microscopy was in 1980, just a fruit waiting to be plucked. I was just lucky to have picked the right area.

  9. People are people. From a science side, I don’t really have a hero. One of the sobering things I feel about winning the Nobel Prize is I’ve had close contact with lots of famous people, scientists, politicians, etc. It’s scary how ordinary most of them seem. They’re not smarter. They’re not more beneficent. They don’t have deeper thoughts. They’re just people. People are people and whether they’re Nobel laureates, the President of the United States, or a billionaire, they’re all people and humans are very limited. Often, when people have success, they tend to forget their limitations, and extrapolate from their success in one area into expertise or proficiency in other areas in which they are not expert. I’m very cognizant of my many limitations, and the only time I’m willing to talk authoritatively is about microscopy because that’s the one area I feel like I’m competent to have an opinion on.

  10. talk to people with different background. How much do you hang out with chemists and biologists and data scientists and mathematicians and so forth? That’s why I always loved Bell Labs, where cell biologists, neurobiologists, computer scientists, material scientists all worked under the same roof and ate lunch together at the same table.

  11. risk, vigor and creativity. Don’t be afraid to take risks. There’s always a way to recover from a failure. Life is short, so live it with all the vigor and creativity you can.
